## Case: case_01_高斯模糊+透明度

- 评判时间：2026-06-08 17:33（北京时间）
- 参与库：canvas2d, css_filter, gsap, pixijs

作为顶级视频特效评判专家，我仔细观看了所有视频，并根据源画面需求和评判维度进行了专业分析。

---

### 各维度排名

| 维度 | 第1 | 第2 | 第3 | 第4 |
| :--- | :--- | :--- | :--- | :--- |
| 效果准确性 | css_filter, gsap | pixijs | canvas2d | |
| 视觉质感 | css_filter, gsap | pixijs | canvas2d | |
| 过渡平滑度 | css_filter, gsap, pixijs | canvas2d | | |
| 细节表现 | css_filter, gsap | pixijs | canvas2d | |

### 综合排名

1.  **第1名：css_filter / gsap** — 两者在视觉效果上几乎无法区分，均完美实现了高斯模糊和透明度叠加的物理参数，并展现出卓越的视觉质感和过渡平滑度。它们利用了浏览器原生渲染的优势，效果自然且高级。
2.  **第2名：pixijs** — 提供了高质量的模糊和透明度过渡，动画流畅。其模糊效果虽与CSS原生滤镜略有不同，但依然保持了很高的专业水准，在细节表现上也令人满意。
3.  **第3名：canvas2d** — 动画时长和透明度变化可能符合要求，但高斯模糊的实现质量不佳，呈现出明显的像素化和块状感，严重影响了视觉质感和细节表现。

### 详细理由

1.  **效果准确性**
    *   **css_filter / gsap (并列第1)**：两者都精确地实现了1.5秒的动画时长，Opacity从100%到30%的线性变化，以及Blur从0px到18px的线性变化。最终的模糊程度和透明度都与需求描述高度吻合，且缓动函数为线性，没有加速或减速感。
    *   **pixijs (第2)**：同样在时长、Opacity和Blur的数值变化上表现准确，动画节奏符合线性缓动。但其高斯模糊的视觉特性与CSS滤镜略有差异，可能在某些细节上与“标准”高斯模糊的感知略有出入，但整体准确性很高。
    *   **canvas2d (第3)**：虽然动画时长和Opacity变化可能符合，但其高斯模糊的视觉效果与18px的预期质量相去甚远。模糊算法的低劣导致无法准确评估其“18px”的视觉呈现，效果准确性大打折扣。
