### Prompt 要点

这是一个**新 Prompt**，不同于 v2 拉片 Prompt（v2 是"看视频→分析画面"，这里是"读文字稿→设计画面"）。

核心指令：

1. **角色**：你是一个顶级的科普视频视觉导演，正在为一篇科普文字稿设计逐秒分镜脚本。
2. **学习 Few-Shot 样本的视觉语法**：不是学表格格式，而是学"持久窗口壳、底部信息栏、鼠标指针、证据截图、高亮强调层、弹性物理动效"这些画面组织方式。参考样本中的设计意图（→ 后面的叙事目的）尤其重要。
3. **输出格式**：与拉片样本完全一致——先"持久画面架构"，再"时间轴分镜表"（四列：时间轴区间 / 图层内容与素材 / 运动与设计意图 / 文字与配音）。
4. **强制约束**：
   - 只使用物理字典描述动画（Scale/Opacity/TranslateX/Y/Rotate/Blur/ClipPath + Linear/Spring/Bezier），不指定任何实现库
   - 每个素材必须标注分类标签（`[BROWSER]`/`[CODE]`/`[STOCK]`/`[AI_GEN]`/`[HUMAN]`）
   - 每段运动必须附带设计意图（用 → 连接叙事目的）
   - 每 3-5 秒至少一次可见 UI 状态变化，禁止单张大卡片承载整句旁白
   - 画面设计必须包含包装层、主体证据层、强调层和辅助运动层
5. **内容方向**：豆包科普视频的视觉主题是"AI 产品 vs 模型"的认知科普。画面应大量使用豆包 App 界面截图（`[BROWSER]`）、对比卡片（`[CODE]`）等具象证据，而非抽象概念图。

### 输出

## 步骤 2：素材清单提取

**执行者**：AI Agent（Claude Code / 其他）

**任务**：从步骤 1 产出的分镜脚本中，提取所有引用的素材，逐个归类、编号、写获取指令。

### 输入

| 文件 | 路径 |
| :--- | :--- |
| 分镜脚本 | 步骤 1 的产出文件 |
| 素材 To-Do 协议 | `2026-06-12_14-05_项目资源导航与工程标准.md` 中"统一素材 To-Do 协议"章节 |

### AI Agent 指令

```
请阅读这份分镜脚本，按照"统一素材 To-Do 协议"的规范，提取所有素材并生成结构化清单。

对每个素材：
1. 分配编号：ASSET_[镜头序号]_[图层序号]
2. 标注分类：GENERATE_BY_CODE / AI_BROWSER_CAPTURE / STOCK_LIBRARY / AI_GENERATED / HUMAN_CAPTURE
3. 写获取指令：
   - [CODE] 类：标注 [AUTO]，无需外部获取
   - [BROWSER] 类：说明需要截取哪个网页的什么状态
   - [STOCK] 类：给出 3 组搜索关键词（中文语义/英文素材/视觉风格约束）
   - [AI_GEN] 类：给出生成 Prompt、风格、构图要求
   - [HUMAN] 类：说明为什么 AI 无法获取
4. 标注当前状态：[AUTO] 或 [TODO: xxx]

输出为 JSON 文件，同时生成一份人类可读的 Markdown 概览。
```

### 输出

| 产物 | 存放路径 |
| :--- | :--- |
| 素材清单 JSON | `07_分镜与素材/素材清单/2026-MM-DD_HH-MM_豆包科普_0-30s_素材清单.json` |
| 素材清单概览 | `07_分镜与素材/素材清单/2026-MM-DD_HH-MM_豆包科普_0-30s_素材清单.md` |

### 人工检查点

- [ ] 素材数量是否与分镜脚本中的图层引用一致？没有遗漏？

## 步骤 3：素材获取

**执行者**：AI Agent（Claude Code / 其他）

**任务**：按素材清单逐个获取素材文件。

### 输入

| 文件 | 路径 |
| :--- | :--- |
| 素材清单 JSON | 步骤 2 的产出文件 |
| 推荐图库工具 | `2026-06-12_14-05_项目资源导航与工程标准.md` 中"推荐图库工具"章节 |

### AI Agent 指令（按素材类型分）

**`[CODE]` 类**：跳过，这些在步骤 4 由 Remotion 代码直接绘制。

**`[BROWSER]` 类**：
```
使用 Playwright 或其他浏览器自动化工具：
1. 访问目标网页
2. 按分镜脚本描述的状态调整页面（如滚动到特定位置、切换到特定标签页）
3. 截图保存为 ASSET_xx_xx.png，存入 07_分镜与素材/assets/
4. 更新素材清单状态为 [DONE]
```

**`[STOCK]` 类**：
```
按候选池工作流执行：
1. 使用素材清单中的搜索关键词，调用 Pexels / Unsplash / Pixabay API
2. 每个素材产出 6-10 张候选，存入 07_分镜与素材/素材候选池/ASSET_xx_xx/candidates.json
3. 候选卡片需包含：thumbnail_url, download_url, source_page_url, provider, author, license, fit_reason, risk_note
4. 等待人工选中后，下载到 07_分镜与素材/assets/ASSET_xx_xx.png
5. 更新 asset_ledger.json
```

**`[AI_GEN]` 类**：
```
使用素材清单中的生成 Prompt，调用图片生成 API。
保存结果到 07_分镜与素材/assets/ASSET_xx_xx.png。
```

**`[HUMAN]` 类**：
```
标记为待人工处理，生成占位图（灰色背景 + 白色文字描述），确保渲染流程不阻断。
```

### 输出

| 产物 | 存放路径 |
| :--- | :--- |
| 素材文件 | `07_分镜与素材/assets/ASSET_xx_xx.png` |
| 候选池 | `07_分镜与素材/素材候选池/ASSET_xx_xx/candidates.json` |
| 素材台账 | `07_分镜与素材/assets/asset_ledger.json` |

### 人工检查点

- [ ] 所有非 `[CODE]` 素材是否都已获取或有占位图？
- [ ] `[BROWSER]` 截图是否截取到了正确的页面状态？
- [ ] `[STOCK]` 候选池是否有足够的选择？选中的图是否与分镜描述匹配？
- [ ] asset_ledger.json 是否完整记录了来源和授权信息？

### AI Agent 指令

```
请基于以下分镜脚本，在现有 Remotion 工程中生成可运行的视频组件代码。

参考现有工程的代码结构：
- src/lib/motion.ts：动效工具函数（linear, smooth, springScale, opacityOut）

- src/composition/DoubaoScience30.tsx：主合成组件

要求：
1. 可以新建组件文件，也可以修改现有文件
2. 充分复用 motion.ts 中的工具函数，需要新增动效原语时扩展该文件
3. 分镜脚本中的物理字典参数直接翻译为 Remotion 的 interpolate/spring 调用
4. 素材文件引用 07_分镜与素材/assets/ 中的实际文件路径
5. [CODE] 类素材在 React 组件中直接用 CSS/SVG 绘制
```

## 迭代循环

完成一轮后，根据步骤 5 的观感评价，回到上游调整：

- **分镜质量不行** → 调整步骤 1 的 Prompt 或 Few-Shot 样本，重跑步骤 1-5
- **素材质量不行** → 调整步骤 3 的搜索策略或手动替换素材，重跑步骤 4-5
- **动效质量不行** → 在步骤 4 中细化指令或手动调整代码，重跑步骤 5
- **Few-Shot 样本本身有问题** → 回到 02_ 拉片阶段，用 v2 prompt 重新拆解或补充更多标杆视频

这就是"改上游 → 看结果 → 判断是否有效"的快速反馈回路。
