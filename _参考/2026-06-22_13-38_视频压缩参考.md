## 上传前视频压缩要求

如果要把本地视频交给 Qwen Code 或类似多模态模型工具读取，不要只看 mp4 原文件大小。Qwen Code 读取本地视频时会转成 base64，体积约膨胀三分之一；例如本次 `01_7634456235240164659_AI集体涨价_480p.mp4` 原文件为 8.71 MiB，但 base64 后约 11.62 MiB，会超过 Qwen Code 本地读取层约 9.9 MiB 的限制。

实际上传前，优先把视频压到 5 MiB 左右。这样 base64 后大约 6-7 MiB，余量比较安全。7 MiB 左右的视频理论上也可能通过，但 base64 后约 9.3 MiB，离限制较近。

## 推荐压缩参数

```bash
/home/fnckc/.local/lib/python3.12/site-packages/imageio_ffmpeg/binaries/ffmpeg-linux-x86_64-v7.0.2 \
  -y -hide_banner \
  -i "输入视频.mp4" \
  -vf "scale=360:-2" \
  -c:v libx265 \
  -preset medium \
  -x265-params log-level=error \
  -b:v 140k \
  -maxrate 180k \
  -bufsize 280k \
  -c:a aac \
  -b:a 32k \
  -ac 1 \
  -movflags +faststart \
  "输出视频_360p_qwenfit.mp4"
```

这组参数的含义是：把竖屏 480p 视频压到宽 360px，高度自动保持比例；视频用 H.265，目标码率约 140k；音频压到 AAC 32k 单声道。它适合"先让模型看懂视频内容"，不是用于最终成片发布。

## 实测结果

| 文件 | 原始大小 | base64 估算 | 判断 |
| :--- | :--- | :--- | :--- |
| 原始 480p 视频 | 8.71 MiB | 11.62 MiB | 超过 Qwen Code 限制，不建议直接上传 |
| 只压音频版 | 7.00 MiB | 9.33 MiB | 画面损失最小，但余量较小 |
| 360p qwenfit 版 | 4.85 MiB | 6.46 MiB | 推荐优先用于 Qwen Code / 模型读取 |
