# doc-grilling — 文档变更追问引擎

**完成标准（硬约束）：** 本 Skill 完成 = `_proposals/P00N_功能名.md` 已写出且状态为 GRILLED。没有写文件就说完成，不算完成。

## 追问规则

一次只问一个问题，给出推荐答案，等用户回答后再问下一个。能从文件直接读出的答案，直接说明并跳过。

## 必须覆盖的五个分支（按序）

**A. 变更起点**
对照依赖关系图的 11 条链，确认：这次改什么？属于哪条链？是内容调整还是路径/结构变化？

**B. Blast Radius（逐文件）**
列出相关链的受影响文件，逐一确认：这个文件要改吗？改什么？有跨链影响吗？

**C. 单一来源**
这次变更会不会让同一信息出现在两个文件里？权威来源是哪个？另一个怎么处理？

**D. 不动的范围**
明确排除不需要改的文件和章节，防止误改。

**E. 完成条件**
用户会用什么方式验证这次变更完成了？

## 完成动作（硬约束）

走完五个分支，用 1-2 段总结，**等用户确认**后执行：

1. `mkdir -p _proposals/`（如果目录不存在）
2. 查看已有 `_proposals/P*.md` 文件，取最大编号 +1（从 P001 开始）
3. 创建 `_proposals/P00N_[功能名].md`，内容如下：

```markdown
# P00N [功能名]

状态：GRILLED

## 变更描述
[用户确认的变更内容]

## 触发链
链 N（+ 链 M）

## 影响文件
- [文件 A]：[改什么]
- [文件 B]：[改什么]

## 不动的范围
- [文件 X]：不涉及

## 单一来源决策
[如有]

## 完成验证方式
[用户说的]
```

写完后说："决策记录已写入 `_proposals/P00N_[功能名].md`。现在调用 `/doc-prd` 继续。"
