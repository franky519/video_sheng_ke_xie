## 问题观察

AI集体涨价拉片拆解文档有三重身份：

1. 它是 v4 Prompt **跑出来的结果**（Step 0 的输出）
2. 它是分镜生成时的 **Few-Shot 样本**（Step 1 的输入）
3. 它**就是未来分镜头脚本应该长成的大体框架**——时间轴 / 底层原片事件 / 后期干预手段 / 音频这个四列表格，是分镜脚本的格式模板

当前依赖关系图只写了"Prompt → 拆解脚本 → 拉片结果"这一条链（链 1），没有写"拉片结果 → 分镜生成"这个格式继承关系。目录导航里也只说"共享工具/提示词"里有 v4 Prompt、"标杆视频库"里有拉片结果，"视频项目"里有分镜脚本——三块目录各自描述，没有一条线把它们串成"这个拆解文档的写法是你未来分镜要模仿的格式"。

更根本的问题在项目总览。它的"全链路生产流程"画的是一条线性流：文字稿 → 分镜 → 素材 → 渲染 → 成片。但这之前缺了一个完整的阶段：怎么知道分镜脚本应该长什么样。真正的流程是：从已有好视频里提取格式 → 用这个格式写新分镜 → 拿分镜去执行。项目总览只写了后半段，没有写前半段（格式是怎么来的、为什么分镜要按这个格式写）。这导致每个新 Agent 都会误把 v4 Prompt 当上游定义点，而不是拆解文档的表格结构。

## 变更意图

在三份文档中串联整条逻辑链路：

1. **项目总览**：把"全链路生产流程"重写为包含完整阶段（阶段零：从好视频学格式 → 阶段一：用格式写新分镜 → 阶段二到五：执行管线）。每个阶段写清楚输入、输出、为什么这么做
2. **目录导航**：在共享工具 → 标杆视频库 → 视频项目三个目录区之间补串联描述
3. **依赖关系图**：链 1 新增一行"拆解文档表格格式 → 分镜脚本格式模板"
