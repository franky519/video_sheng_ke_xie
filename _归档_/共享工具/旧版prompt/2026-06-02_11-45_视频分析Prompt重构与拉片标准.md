# 视频分析 Prompt 重构与程序化拉片标准

> **最后更新时间**：2026-06-05 11:05（北京时间）  
> **更新次数**：第 3 次  
> **版本**：v2.0（图层解耦与程序化渲染版）

本指南确立了利用 AI 对视频进行拉片分析时的标准提示词（Prompt）和输出规范。其终极目标是让 AI 产出**结构化、物理量化且面向编程实现**的代码逻辑，从而能让其他 AI（或渲染脚本）直接利用代码（如 Python 的 Moviepy/OpenCV 或 Node.js 的 Remotion 库）全自动复刻视频风格，避免人工 AE/PR 操作造成的逻辑黑盒。

---

## 一、 重构前后的核心变化差异

如果用“前情卡片导入与点击（00:04 - 00:07）”这段画面来对比，重构前后的输出差异如下：

### 1. 画面与运动彻底解耦
*   **重构前（混合描述）**：将背景、卡片和鼠标指针全部放在一起混合描述。信息糅杂，程序化解析时难以提取独立的轨道。
*   **重构后（图层独立轨道）**：将画面严格按照 `Layer ID` 垂直分割：
    - **Layer 0 (底景)**：论坛实拍 B-roll。
    - **Layer 1 (悬浮卡片)**：自制圆角阴影卡片。
    - **Layer 2 (鼠标指针)**：像素风 PNG 鼠标。
    每一个图层在“素材列表”里声明资产，在“运动物理参数”里声明各自的时间轴动效，完全对齐编程中的 `CompositeVideoClip` 或 React 图层组件结构。

### 2. 制作手法走向“程序化渲染实现”
*   **重构前（人工软件词汇）**：提及“使用 AE 关键帧、应用 PR 高斯模糊”等，这些都是手工操作工具，代码库无法直接转化。
*   **重构后（代码库与数学函数）**：
    - 针对 Layer 0 模糊：指明使用 OpenCV 的 `cv2.GaussianBlur` 算法或 CSS 的 `filter: blur()`。
    - 针对 Layer 1 缩放：指明使用基于时间 `t` 的缩放插值，或者 Remotion 中的 `spring` 弹性物理引擎。
    - 针对 Layer 2 移动：提供起止点在屏幕上的百分比坐标 `[x%, y%]`，直接对应代码里定位参数。

## 二、 重构后的完整 Prompt 模板

新 Prompt 已保存至本地：`2026-06-02_11-45_Gemini物理拉片分析prompt.txt`。其内容规范如下：

```text
你是一个顶级的视频程序化渲染专家与视频算法工程师，正在进行专业的视频“拉片”分析。请对该视频的前 30 秒（00:00 - 00:30）进行程序化视器的图层解耦与关键帧物理参数拆解。

为了能够让其他 AI 完全通过代码（如 Python 的 Moviepy/OpenCV，或 Node.js 的 Remotion 等）来实现视频风格的无缝编程复刻，你的分析必须遵循以下两条硬性铁律：

1. 【画面元素与运动动效彻底解耦，按图层（Layer ID）组织】
   - 严禁将不同的素材和动效混在一起口语化描述。你必须将画面拆解为多个独立的渲染图层（如 Layer 0 衬底背景、Layer 1 悬浮卡片、Layer 2 鼠标指针）。
   - 【图层解耦与素材列表】：对每个图层，定义其“素材资产属性”：
     * 判定素材类型：【网页/文档截图】（说明对应新闻主题）、【复合拼贴元素】（包含哪些独立抠图Logo层）、【第三方实拍B-roll】、【自制文本卡片】。
     * 严禁描述无关紧要的衣服、长相、色彩感觉等非结构性细节，只写该素材的物理属性（如圆角、投影、原始分辨率等）。
   - 【图层运动物理参数】：对每个图层，在对应的时间轴区间内，给出该图层独立的渲染关键帧和物理运动参数，包括：
     * 坐标变化：屏幕相对坐标起止点（如 [x%, y%]，以屏幕左上角为 [0%, 0%]）。
     * 尺度变化：Scale 缩放比例（如在 0.5s 内从 0% 弹性放大至 100%）。
     * 特效参数：不透明度（Opacity 从 0% 到 100%）、高斯模糊半径（Blur Radius）的具体数值渐变过程。

2. 【“程序化制法”拒绝人工软件词汇，提供代码库渲染逻辑】
   - 在“程序化制法”一栏中，彻底抛弃 AE 关键帧、PR 剪辑等手动工具词汇。
   - 必须提供具体的编程实现逻辑和对应的代码库参考（如 Moviepy 的 CompositeVideoClip、OpenCV 的 GaussianBlur、Remotion React 组件中的 spring 弹性动画、FFmpeg 滤镜参数等）。

请以 Markdown 表格的形式输出，表格列规定如下：
| 时间轴区间 | 图层解耦与素材列表 | 图层运动物理参数 | 文字与配音内容 | 程序化制法与代码库参考 |
```
