# 豆包科普视频前 30 秒程序化动效样片实现计划

本计划旨在通过前端技术（HTML/CSS/JS + GSAP）开发一个高保真的 30 秒动效样片，完美复刻《豆包科普视频：0~30秒图层运动工程调整与待办脚本》中所定义的画面图层与物理动效。由于本地缺少 FFmpeg 且 Moviepy 未安装，采用浏览器渲染结合 GSAP 时间轴的方案是最优解，不仅能完美还原弹性动画（Spring Easing）和高斯模糊等特效，还能通过浏览器会话录制输出 WebP 动效视频。

最后更新时间：2026-06-05 11:05（北京时间）
更新次数：第 2 次

## 待确认与设计细节

* 旁白配音同步：由于目前为无声渲染，我们将通过底部滚动字幕的形式，将“关联旁白源”中的文案与动画节奏进行毫秒级对齐。
* 3D 红色问号图标：我们将在 SVG 中直接绘制一个带渐变、红色立体厚度和呼吸阴影的矢量问号，替代外部图片资产，以保证极佳的分辨率和动效。

## 方案设计

### 1. 动效引擎与技术栈选择
* **基础框架**：单文件 HTML (`index.html`) + 外部 CSS 样式 + Vanilla JS 逻辑。
* **动画库**：通过 CDN 引入 GSAP 3.x 库，利用其 `gsap.timeline` 管理 0-30 秒的全局关键帧。GSAP 具有极强的属性插值能力，能轻松模拟 `damping` 和 `mass` 组成的弹性效果。
* **物理字典映射**：
  - 位置坐标：CSS absolute 定位配合百分比（`top`, `left`）。
  - 缩放与位移：`transform: scale() translate()`。
  - 模糊滤镜：`filter: blur()`。
  - 擦除遮罩：`clip-path: inset()`。

### 2. 界面与素材的代码绘制实现
为了避免导入缺失 of PNG 素材导致黑屏或报错，所有资产将以代码生成（`GENERATE_BY_CODE`）形式高保真绘制：
* **ASSET_01_01 (iPhone 手机外壳)**：使用 SVG 绘制精致的圆角刘海屏手机外壳，带金属高光和质感投影。
* **ASSET_01_02 / ASSET_01_03 (聊天界面)**：用 CSS Grid/Flexbox 模拟精美流畅的聊天对话流，普通模式使用无排版灰色气泡，专家模式使用卡片化、带橙绿高亮的高对比气泡。
* **ASSET_02_01 / ASSET_02_02 / ASSET_03_01 (卡片类)**：全 CSS 样式化，配以柔和投影、精致圆角与排版。
* **ASSET_03_02 (放大镜)**：通过圆形 `clip-path` 容器在相同位置叠放一个缩放至 250% 的聊天 DOM 副本，并施加正弦抖动。

### 3. 分段关键帧对齐表
我们将严格按照 05 动效脚本中的时间轴控制 GSAP 时间线：
* **0s - 5s**：左侧普通手机 A 与右侧专家手机 B 延迟 1 秒后依次弹性弹出，接着手机 A 淡出至 40% 不透明度，手机 B 缓慢微幅放大至 105%。
* **5s - 15s**：手机淡出。用户提问框淡入并向上平移；随后平庸回答卡片淡入，其文字向上匀速滚动 80px，最后模糊淡出。
* **15s - 25s**：专家回答卡片弹性放大入场，绿、橙高亮色块依次从左向右擦除展开，放大镜在指定坐标缩放弹出并进行轻微抖动。
* **25s - 30s**：提示词卡片 A、B 分别自左右两侧倾斜滑入，中心红色立体问号爆炸式缩放弹出并产生红色发光阴影。

## 验证与输出计划

### 自动化录制与输出
1. 启动本地 Python 轻量级服务：`python3 -m http.server 8000`  
2. 启动浏览器 subagent 访问 `http://localhost:8000/index.html`，播放并停留 30 秒以捕获完整的动效录像。
3. 录像自动生成的 WebP 视频将存入 artifacts 目录供直观预览。

### 手动交互验证
* 页面中将内置一个控制面板，包括：播放/暂停按钮、进度条滑块、时间显示器。你可以通过拖动滑块或点击暂停，随时在浏览器中查看任意时间点的画面细节和图层状态。
