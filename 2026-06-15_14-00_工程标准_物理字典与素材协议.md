## 防偏差工程标准：闭环物理动作字典

为了防止大模型在分析视频或输出动效时凭空捏造（Hallucinate）不存在的、或者不兼容的第三方库（如 Moviepy/Remotion）私有 API 导致其他 AI 编写代码时报错，我们规定所有"运动与渲染"的描述必须翻译为**"标准网页底层物理字典"**。

在提示词与脚本中，只允许大模型映射至以下通用属性：

### 1. 空间定位（基准百分比）
* 所有的位置坐标一律以左上角为基准点 `[0%, 0%]`，右下角为 `[100%, 100%]`。
* 严禁直接使用设备相关的固定像素值（如 `1920x1080` 下的特定像素点），必须写成相对百分比坐标，例如 `位置：[50%, 50%]`（居中）。

### 2. 标准物理变换字典（CSS/JS Primitives）
动效拉片中对"运动"的程序化描述，必须直接映射到如下标准物理量：
* `Scale (缩放)`：以基准值 `100%` 进行变化。
* `Opacity (不透明度)`：取值范围 `0%`（完全透明）至 `100%`（完全可见）。
* `TranslateX / TranslateY (相对平移)`：用相对窗口宽高的百分比表示（例如：`TranslateX 从 100% 移至 0%` 表示自右滑入）。
* `Rotate (角度旋转)`：用顺时针角度度数表示（例如 `rotate(-15deg)`）。
* `Blur (高斯模糊)`：用模糊像素半径表示（例如 `blur(15px)`）。
* `ClipPath (裁剪遮罩)`：采用 CSS 的矩形滑块语法（例如 `clipPath: inset(0% 100% 0% 0%)` 表示水平方向向右擦除露出）。

### 3. 标准插值与缓动控制（Easing & Springs）
* **线性插值（Linear）**：即在指定秒数内匀速变化。
* **弹性物理函数（Spring）**：必须提供两个关键控制量：
  * `Damping (阻尼系数)`：一般介于 `10 - 20` 之间，越低越弹，通常 `12` 为经典回弹，小于 `10` 容易产生反复震荡。
  * `Mass (质量系数)`：一般为 `0.5 - 1.0`，越低越轻，弹出速度越快。
* **贝塞尔曲线控制（Bezier）**：必须指定起止控制点参数（例如 `cubic-bezier(0.25, 0.1, 0.25, 1.0)`）。

## 资产追踪工程标准：统一素材 To-Do 协议

为了把抽象的"镜头描述"彻底转为工程上可被 AI 检查和人类核对的待办清单，脚本中引用的所有非代码生成资产，都必须遵循以下**资产 To-Do 协议**。

### 1. 资产分类与命名规范
每个素材编号为 `ASSET_[镜头序号]_[图层序号]`（如 `ASSET_01_02.png`），统一存入 `07_分镜与素材/assets/`。

| 素材分类 | 生产方式 | 状态标记 |
| :--- | :--- | :--- |
| **`GENERATE_BY_CODE`** | 纯代码绘制（CSS/SVG/渐变背景），无外部依赖 | `[AUTO]` |
| **`AI_BROWSER_CAPTURE`** | AI 通过浏览器自动化访问实际网页并截取（如 ChatGPT 界面、新闻页面） | `[TODO: BROWSER]` |
| **`STOCK_LIBRARY`** | 搜索引擎/图库检索的公共素材（实拍 B-roll、图表等） | `[TODO: STOCK]` |
| **`AI_GENERATED`** | 提示词驱动 AI 生成的图片（如 Midjourney），需记录 Prompt | `[TODO: AI_GEN]` |
| **`HUMAN_CAPTURE`** | 确实只有人类才能获取的素材（极少数情况） | `[TODO: HUMAN]` |

### 2. AI 自动化执行规则

1. **自动归网与命名**：AI 获取或生成资产后，必须重命名为对应资产 ID 并存入 `07_分镜与素材/assets/` 目录。
2. **状态自动勾选**：图片就绪后，将对应文档中的 `[TODO: ...]` 修改为 `[DONE]`。
3. **未就绪告警**：渲染时如发现 `[TODO]` 状态的资产文件不存在，脚本必须阻断并告警。
4. **草稿隔离**：AI 原始输出必须原样保存在对应的归档目录中，人工校正在独立文件中进行，防止源头污染。

### 3. 素材候选池工作流

每个 `[TODO: STOCK]` 素材的候选池存放于 `07_分镜与素材/素材候选池/ASSET_xx_xx/`：

| 字段 | 作用 |
| :--- | :--- |
| `asset_id` | 对应 `ASSET_xx_xx`，防止图片和镜头错位。 |
| `thumbnail_url` | 用来快速预览 6-10 张候选。 |
| `download_url` | 选中后用于下载原图或合适尺寸图。 |
| `source_page_url` | 以后追溯来源和授权。 |
| `provider` | Pexels / Unsplash / Pixabay / Openverse / Brave 等。 |
| `author` | 署名需要。 |
| `license` | 授权检查需要。 |
| `fit_reason` | AI 说明为什么它适合这个镜头。 |
| `risk_note` | 比如"可能有商标""授权需复核""画幅太窄"。 |

### 4. 素材台账

最终选定的素材在 `07_分镜与素材/assets/asset_ledger.json` 中记录完整的来源和授权信息，字段包括：`asset_id`、`source_url`、`provider`、`license`、`author`、`fetched_at`、`status`。

### 5. 推荐图库工具

| 工具 | 优先级 | 适合用途 |
| :--- | :--- | :--- |
| Pexels API | 第一优先 | 实拍照片、人物、办公、科技感 B-roll |
| Unsplash API | 第二优先 | 高质量摄影图、氛围图 |
| Pixabay API | 补充 | 插画、图标、透明感素材 |
| Openverse API / Wikimedia Commons | 补充 | 公共领域、CC 授权、历史/科学图 |
| Brave Image Search API | 补充 | 全网灵感，授权需人工复核 |

不推荐：Google AI Mode（无公开 API）、Bing Image Search API（已退役）、Google Programmable Search（已停止新客户）。
