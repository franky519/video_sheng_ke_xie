## 素材标签

拉片分析和分镜生成时，AI 需要对画面里的每样东西打一个标签——这个东西本质上是屏幕录屏还是后期加的文字卡片？标签是后续所有管线决策的基础：哪些素材要截取网页、哪些纯靠代码画、哪些得用图库找。

### 拉片素材标签

v4 拉片 Prompt 强制使用的五类标签。你在分析差评君/冲浪Plus 视频时，看到的每一层画面都要归到这五类之一：

| 标签 | 是什么 | 典型场景 |
| :--- | :--- | :--- |
| `[SCREEN_RECORD]` | 连续的真实屏幕操作录屏 | 比如一段差评君打开豆包 App、翻看聊天记录的连续录屏。画面里鼠标在移动、页面在滚动——这些内部动作用流媒体语法描述（见下方流媒体素材语义），别用 Translate |
| `[FOOTAGE]` | 连续实拍视频片段或 CG 动画 | 比如一段工厂流水线的实拍、一段 TVC 级别的三维渲染动画。和 `[SCREEN_RECORD]` 一样是连续视频流，但内容不是屏幕操作而是实拍或 CG |
| `[CODE]` | 后期软件生成的 UI 元素 | 高亮框、文字卡片、弹窗、进度条、遮罩——这些不是原视频里的东西，是后期加上去的。第 4 步渲染时直接靠代码绘制，不需要外部素材 |
| `[STOCK]` | 后期插入的静态图片、插画、空镜 | 比如科普段落里配一张芯片特写照片、一段科技感的 B-roll 空镜。从图库 API（Pexels/Unsplash）获取 |
| `[BROWSER]` | 静态网页或 App 截图 | 比如豆包 App 的聊天界面截图——这种截图是通过浏览器自动化访问真实页面截取的，不是图库找的 |

如果你搞混了 `[SCREEN_RECORD]` 和 `[BROWSER]`：把一段动态录屏标成了 `[BROWSER]`——后面素材获取阶段 AI 会去截图而不是保留视频流，整个画面的动态感就丢了。反过来把一个静态网页截图标成了 `[SCREEN_RECORD]`——渲染时会试图播放一个不存在的视频流，导致空白画面。

分镜生成和素材清单阶段额外使用的 `[AI_GEN]`（AI 生成图片）和 `[HUMAN]`（人工获取）这两个标签，描述的是素材怎么获得，不是画面物理本质，不在拉片标签范围内。

## 物理字典

所有运动描述只能用下面这些底层物理量来表达，不能出现“用 GSAP 做一个 bounce 动画”、“调用 spring 库”这种特定于某个库的写法。目的是让渲染这一步的 Agent 拿到分镜脚本后，自己决定用哪个库去实现这些物理量——今天用 GSAP 实现，明天换 Motion 也不用改分镜。

### 空间定位约束

任何画面元素的位置，用左上角 `[0%, 0%]` 到右下角 `[100%, 100%]` 的相对百分比来描述。

比如你想描述“一个文字卡片出现在屏幕正中央偏上的位置”——写 `X: 50%, Y: 30%`，表示卡片左上角在画面宽度一半、高度 30% 的位置。不管最终视频是 1920x1080 的横屏还是 1080x1920 的竖屏手机，卡片都在那个相对位置。

如果你写成 `left: 960px, top: 324px`，在 1920x1080 的画布上确实居中偏上，但在手机上看就跑飞到右上角去了——因为这个绝对像素值相对于手机屏幕太大了。永远用百分比，不用固定像素。

### 图层变换参数

一个普通图层自身能做的六种底层变换。设初始状态，然后指定终点状态和缓动方式——这就是一个完整动画的描述了。

**`Scale`（缩放）**：基准值 `100%`。弹窗出现效果通常写成“Scale 从 70% 到 100%”，表示从小放大弹出。数值可以超过 100%，比如背景图做缓慢推近写 `Scale 100% → 110%`。

**`Opacity`（不透明度）**：`0%` 完全透明（看不见），`100%` 完全可见。图层渐隐：`Opacity 100% → 0%`。图层叠化转场本质就是两个图层同时做反向淡变。

**`TranslateX / TranslateY`（相对平移）**：用相对窗口宽高的百分比表示移动距离。一个浏览器窗口从屏幕下方滑入：`TranslateY: 100% → 0%`（从画面下方完全移入到最终位置）。注意这描述的是容器本身在动——别用它描述录屏里网页自己滚动的内容，那是 Scroll 的范畴（见下方流媒体素材语义）。

**`Rotate`（旋转）**：用顺时针角度表示。比如你想让一张截图旋转 3 度模拟手持拍摄的晃动感，写 `Rotate 3° clockwise`，不要写 `transform: matrix(0.97, 0.26, -0.26, 0.97, 0, 0)` 那种矩阵调用。如果你写 matrix——后面渲染 Agent 看不懂你想干嘛，可能会省略这个效果或给出完全错误的旋转值。

**`Blur`（高斯模糊）**：用像素半径表示。背景图做毛玻璃写 `Blur: 8px`；模拟浅景深虚化写 `Blur: 24px`。想反向（从模糊变清晰）：`Blur 24px → 0px`，就是焦点逐渐对上的效果。

**`ClipPath`（裁剪遮罩）**：采用 CSS 矩形语法 `inset(上 右 下 左)`。高亮框从中间向两边展开：`inset(0% 50% 0% 50%) → inset(0% 0% 0% 0%)`——因为初始状态裁剪掉左右各 50%，终点全部展开。也可以用圆形 `circle()` 做圆形遮罩展开效果。

**`Pop-up`（弹性弹出）**：这不是一个新物理量，而是 `Scale` + `TranslateY` + `Spring` 缓动的组合。比如“通知气泡从下弹出”——本质是 `Scale 70%→100%`、`TranslateY 10%→0%`，配 Spring 缓动。别写成调用某个弹窗库，那会在不同渲染框架间没法翻译。

### 缓动控制

运动的节奏感——同样的位移，匀速走和先快后慢走是完全不同的观感。

**`Linear`**：指定秒数内匀速变化。比如“进度条 3 秒内匀速走完”——它是一个标准的、不加速不减速的机械运动，适合进度条、倒计时这类需要精确对应的场景。

**`Spring`**：模拟弹簧的物理弹性，必须提供 `Damping`（阻尼，控制弹性衰减速度）和 `Mass`（质量，控制回弹惯性强弱）。常用 Damping 10-20、Mass 0.5-1.0。比如“文字卡片弹入画面”——Damping 15、Mass 0.8，卡片先是超调到目标位置外一点，再回弹稳定。Damping 越小弹性越大、Mass 越大惯性越大——想一个按钮像果冻一样弹出来就降 Damping 到 8，想它稳稳到位就升到 20。

**`Bezier`**：必须指定四个控制点参数，常用 `cubic-bezier(0.25, 0.1, 0.25, 1.0)`——这是先快后慢的自然曲线。元素优雅地淡入、平滑地滑入大部分都用这个值。

### 流媒体素材语义

`[SCREEN_RECORD]` 和 `[FOOTAGE]` 是连续视频流——它们容器本身可以 Translate 滑入滑出，但容器内部画面的动作有一套专用语义。核心原则：分清“是容器在动”还是“容器里的内容在动”。

**`Playback`（内部播放）**：素材画面里正在发生的客观动作。人在走路、镜头在推进、车辆在移动——这些都是录进画面里的原始内容，写 `Playback` 表示“这是素材自带的，不是我控制它动的”。

**`Scroll`（视口滚动）**：专用于录屏素材，描述界面视口上下滑动。比如差评君滚动浏览一个长网页——这是 `Scroll`，不是 `TranslateY`。如果你写成 `TranslateY`——渲染时会试图把整个录屏容器往上平移，但录屏内部的网页并没有真的滚动，观众看到的是一段静态截屏在平移而不是一段正在被浏览的网页，整个信息密度就塌了。

**`Cursor Move`（光标滑动）**：描述鼠标或手势指针从 A 点移到 B 点。和 `Scroll` 的区别：光标移动时视口不滚——比如鼠标滑过菜单项高亮它们，屏幕没有翻页。

**`Typewriting`（拟真打字）**：输入框逐字输入、拼音候选栏出现、光标闪烁。比如“用百度搜索 AI 芯片”——输入框里逐字出现“AI 芯片”，这是 `Typewriting`；如果是后期加的文字卡片逐字出现，那是容器的 ClipPath 或 Opacity 变化，不是 Typewriting——因为画面里没有真实的输入框。

**`Click`（点击反馈）**：按钮被点击后的页面响应、涟漪动画、选中高亮。比如“点了一下搜索按钮，页面跳转到结果页”。

一条规则记住这些的边界：容器本身的出入场（窗口从下方滑入、卡片从左边飞入）写 `Translate`；容器内部录屏的动作（网页滚动、打字、点击）写流媒体语法。一个场景描述里两种会同时存在——“浏览器窗口 TranslateY 滑入，滑入后窗口内部 Scroll 浏览搜索结果”。

### 运镜与转场

#### 静态 B-roll 默认运镜（Ken Burns）

当 `[STOCK]` 或 `[BROWSER]` 的静态图片作为全屏或接近全屏的 B-roll 使用时，一张静态照片停 3 秒会像 PPT。默认给它做一个缓慢的推拉摇移：

`Ken Burns Effect`（缓慢推拉摇移）——也就是 `Push-in`（缓慢推近）、`Pull-out`（缓慢拉远）、或 `Slow Pan`（缓慢平移）。幅度一般 5%-10%，在整段 B-roll 持续时间内匀速完成。比如“芯片特写照片 3 秒，Scale 100% → 105%，制造缓慢推进的呼吸感”。

只有确实出现多层素材以不同速度移动时（比如前景和背景分成两层，前景平移快、背景平移慢），才额外说明 `Parallax`，否则不需要提。

#### 场景转场术语

两个场景之间怎么更迭——不能只写“上一个画面消失，下一个画面出现”，必须用专业剪辑术语。当前 v4 拉片 Prompt 强制使用以下四种：

| 术语 | 含义 | 典型场景 |
| :--- | :--- | :--- |
| `Hard Cut`（硬切） | 瞬间替换画面，没有过渡 | 最常用转场。上一帧是画面 A，下一帧直接是画面 B。节奏快的科普视频里 90% 都是硬切 |
| `Cross Dissolve`（叠化） | 两个画面透明度交叉渐变——旧画面从 100% 淡到 0%，同时新画面从 0% 淡到 100% | 适合温暖或怀旧的氛围切换，比如从产品界面切到用户生活场景 |
| `Push`（推像） | 新画面把旧画面顶出视窗——通常新画面从右侧推入，旧画面被推到左侧移出 | 有明确的推进感和空间感，适合“接下来讲另一个方面”的段落切换 |
| `Masking`（遮罩） | 利用遮罩区域或画面中某个运动物体的边缘引出新画面——比如一个圆形从画面中央放大，里面是新画面 | 讲究视觉巧妙的转场，比如用鼠标指针划过带出一道新画面 |

如果你把 Cross Dissolve 写成了“画面淡出，然后下一画面淡入”，渲染 Agent 可能会做成先黑屏再亮起（中间有个黑帧）而不是真正的叠化——这两个画面要同时存在且同步淡变才算叠化。Masking 如果写成“加个圆形遮罩”——需要说明遮罩是从哪出现的、借助了画面里的什么元素做引子。

历史拉片里可能出现的 `Fade to/from Black`、`Glitch`、`Match Cut` 等术语可以作为补充理解，但 v4 拉片优先按上表四种归类。

## 资产协议

脚本里引用的所有非代码生成的资产都必须进入素材待办清单，AI 和人类各司其职：AI 能获取的自己取，获取不了的就标给人类。

### 资产分类与命名规范

每个素材编号为 `ASSET_[镜头序号]_[图层序号]`，存入 `视频项目/.../04_素材清单/assets/`。

| 素材分类 | 怎么获得 | 状态标记 |
| :--- | :--- | :--- |
| `GENERATE_BY_CODE` | 纯代码绘制——一个高亮框、一段进度条、一个文字卡片。不用任何外部文件，渲染时直接在组件里画 | `[AUTO]` |
| `AI_BROWSER_CAPTURE` | AI 打开浏览器访问目标网页，调整到需要状态后截图 | `[TODO: BROWSER]` |
| `STOCK_LIBRARY` | 从 Pexels / Unsplash / Pixabay 等图库 API 检索 | `[TODO: STOCK]` |
| `AI_GENERATED` | 通过提示词调用图片生成 API 产出 | `[TODO: AI_GEN]` |
| `HUMAN_CAPTURE` | 只有人类才能获取——比如需要登录系统查阅的内部数据界面、需要签字才能使用的资料 | `[TODO: HUMAN]` |

### AI 自动化执行规则

1. AI 获取或生成资产后，必须重命名为对应资产 ID（如 `ASSET_01_03.png`）并存入 `视频项目/.../04_素材清单/assets/`。
2. 资产文件就位后，把对应文档中的 `[TODO: BROWSER]` 等标记改为 `[DONE]`。
3. 渲染时如果发现任一 `[TODO]` 状态的资产文件不存在，脚本必须阻断并告警——不能拿错误截图或占位图糊过去。
4. AI 原始输出（API 返回的原图、原始 HTML 截图）必须原样保存在对应的归档目录里。人工校正在独立文件里进行。不要在原始输出上直接改——一旦污染源头，后面所有对比和追溯都失效了。

### 推荐图库工具

按优先级排——第一个搜不到再用第二个，依次往下：

| 工具 | 优先级 | 适合找什么 |
| :--- | :--- | :--- |
| Pexels API | 第一优先 | 实拍照片：人物、办公场景、科技感 B-roll。免费、质量稳定 |
| Unsplash API | 第二优先 | 高质量摄影图、氛围图。构图讲究，适合做背景和空镜 |
| Pixabay API | 补充 | 插画、图标、透明背景素材。Pexels/Unsplash 以摄影为主，需要矢量感和图标时用这个 |
| Openverse API / Wikimedia Commons | 补充 | 公共领域、CC 授权的历史和科学图。科普项目中引用历史照片、科学图示时这里更合适 |
| Brave Image Search API | 补充 | 全网灵感搜索。素材丰富但授权需要人工逐张复核，只在以上四个都找不到时用 |

不推荐：Google AI Mode（无公开 API，无法程序化调用）、Bing Image Search API（已退役）。
