## 这个文件有什么用

本文件是"省科协科普视频"项目的**结构性依赖地图**。它把项目里所有关键文档之间的依赖关系显式写出来——谁引用谁、谁的定义被谁消费、改一个文件会波及哪些文件。

四个核心入口文档（项目总览、目录导航、工程标准、端到端管线手册）的 IMPORTANT 区块里都引用了本文件。任何一个 AI session 在修改项目文件之前，读完核心入口文档后，接着读本文件，就能自动算出完整的 blast radius——而不需要人类记住"改 v4 Prompt 还要改工程标准的素材标签那一章"。

读本文件时配合 **Proposal 工作流** [计划中]（`PROPOSALS.md` 中的结构性变更流程，当前尚未落地）使用：你先提一个 proposal 说想改什么，AI 拿本文件算出影响范围写到 Impact 段落，跟你确认后再按 Change Plan 逐个文件对齐。

## 全局路径约定

本文件中出现的所有文件名和脚本名（如 `openrouter_segment_analyze.py`、`旁白稿.md`、`完整分镜头脚本.md`）均为**简化标识**，不携带时间戳前缀。实际精确文件名和当前入口路径请先查询[目录导航](./2026-06-17_17-57_目录导航.md)。做 `rg` 或路径匹配时，用简化标识匹配即可，AI 应先从目录导航获取当前精确路径再操作文件。

## 一、核心入口文档的四角关系

这四个文档是项目的地基。每个文档的 IMPORTANT 区块都互相引用，形成闭环。

```
项目总览 ←→ 目录导航
   ↕           ↕
工程标准 ←→ 端到端管线手册
```

**职责边界（不交叉）：**

| 文档 | 管什么 | 不管什么 |
|------|--------|---------|
| 项目总览 | 愿景、生产流程、当前进度、缺口、下一步 | 不管理目录结构和文件路径索引 |
| 目录导航 | 目录结构、文件入口、Prompt/脚本/视频产物索引、引用维护规则 | 不描述管线步骤和 Agent 指令 |
| 工程标准 | 运动描述约束、素材标签定义、素材资产生命周期、推荐工具 | 不描述"先做哪步后做哪步" |
| 端到端管线手册 | 从文字稿到成片的步骤 0-5 完整操作指南、每步的输入输出路径和 Agent 指令 | 不重复定义物理字典和素材标签 |

**关键约束：** 同一个信息只能存在于四个文档中的**一个**里。如果发现工程标准里定义了素材标签、管线手册里又定义了一遍——必须在一次 proposal 里合并到一个文档，另一个文档改为引用。

### 链 1：Prompt 版本升级（v3 → v4，或 v4 → v5）

这是最复杂的 blast radius，因为 Prompt 是整条拉片管线的上游定义者。

**起点：** 新增或修改 `共享工具/提示词/Gemini工业级原片后期解耦拉片分析prompt_vN.txt`

| 受影响文件 | 原因 | 具体要改什么 |
|-----------|------|-------------|
| **工程标准** | v4 Prompt 里强制了素材打标（5 类）、转场类型（4 种）、原片/后期解耦等要求——工程标准里"拉片素材标签"和"场景转场术语"两章是这些要求的**镜像定义**。Prompt 改了标签类型或术语定义，工程标准必须同步 | 素材标签章节、转场术语章节、可能还有流媒体语法和物理字典的新增项 |
| **端到端管线手册** | 步骤 0 里硬编码了 prompt 文件路径和 v4 要求表格。Prompt 改版本号或改核心要求，手册里的路径和要求表必须同步 | 步骤 0 的 v4 Prompt 核心要求表格、输入/输出路径 |
| **项目总览** | "Few-Shot 上游：标杆视频拉片"段落写了当前使用 v4、v4 的核心变化是什么、有哪些标签要求。Prompt 换代时总览必须反映新版本号、新能力、新要求 | v4→vN 的版本号、核心变化描述、拉片要求表格 |
| **目录导航** | "共享工具/提示词"章节按状态列出了 prompt 文件（当前入口 vs 历史版本）。新 prompt 要加入口，旧 prompt 要移位到历史版本 | 提示词表格 |
| **所有拉片脚本** | `openrouter_segment_analyze.py`、`gemini_native_analyze_30s.py` 等脚本内部硬编码了 prompt 文件路径。改路径或文件名时脚本必须同步 | 脚本内的 prompt 文件路径 |
| **已有拉片结果的备注** | C、D 两个标杆拉片是 v2 产物，项目总览标注了"仍可作为临时样本"。如果 v5 引入 incompatible 的变化（比如格式全换），需要在总览和导航里更新这些旧样本的状态和是否仍可用的判断 | 项目总览的 Few-Shot 缺口表 |
| **补充提示词文件** | `共享工具/提示词/完整视频连续追问分段拉片补充提示词.txt` 等补充 Prompt 变化时，其依赖的版本假设、标签体系、输出格式约束也应同步审查 | 补充 Prompt 中的版本引用和标签定义 |
| **连续追问实验脚本** | `openrouter_fullvideo_followup_probe.py` 通过 `--prompt-file` 加载补充提示词。主 Prompt 或补充 Prompt 改版本号/文件名——脚本的命令行使用说明和工程计划里的示例命令要同步 | 脚本使用说明和工程计划中的 `--prompt-file` 引用路径 |

**不在影响范围内：**
- 分镜脚本本身（分镜用的是"分镜生成 Prompt"而非拉片 Prompt，两个是不同的 Prompt）
- 素材清单（素材清单是从分镜出来的，不直接碰拉片 Prompt）
- 渲染工程代码

### 链 2：工程标准（物理字典或素材协议）变更

**起点：** 修改 `工程标准_物理字典与素材协议.md`

| 受影响文件 | 原因 | 具体要改什么 |
|-----------|------|-------------|
| **端到端管线手册** | 步骤 1（分镜生成）的 AI Agent 指令里引用了"物理字典标准"作为运动描述约束；步骤 2（素材提取）引用了"统一素材 To-Do 协议"；步骤 3（素材获取）引用了"推荐图库工具"。工程标准改了任何一章，手册里对应的引用约束就自动变了 | 步骤 1/2/3 中引用工程标准的段落 |
| **所有分镜脚本** | 分镜脚本的生成规则是"只使用物理字典描述动画"。如果物理字典新增或删除了某个物理量（比如新增 `Skew`、删除 `ClipPath`），已有分镜和将来生成的分镜都受约束 | 已有分镜中的运动描述可能需要审查是否仍符合新版字典 |
| **所有素材清单** | 素材清单的分类体系（`GENERATE_BY_CODE` / `AI_BROWSER_CAPTURE` 等）直接对应工程标准的"资产分类与命名规范"。如果分类增加或改名，清单结构必须跟上 | 素材清单 JSON 的分类字段和 Markdown 概览 |
| **渲染工程** | 工程标准里的图库优先级（Pexels > Unsplash > Pixabay）影响步骤 3 的素材选择策略，从而间接影响渲染工程里引用的素材 | 非直接影响，但如果素材来源变了，渲染工程里的 public/assets symlink 可能需要更新 |
| **v4 Prompt** | 工程标准里的"拉片素材标签"（5 类）和"场景转场术语"是 Prompt 中强制要求的镜像。如果工程标准改了标签定义，Prompt 文件本身可能也要改——这是双向的 | Prompt 中的标签要求和转场类型列表 |
| **项目总览** | 总览的"Few-Shot 上游"段落引用了 v4 Prompt 的素材打标和原片/后期解耦要求——这些要求是从工程标准的物理字典约束中衍生出来的 | 可能需要同步更新总览中的相关描述 |

### 链 5：素材清单变化

**起点：** 修改或重新生成 `04_素材清单/` 下的 JSON 和 MD 文件

素材编号（`ASSET_xx_xx`）是本链条的关键锚点。编号一变，以下五个渲染工程内的具体文件必须逐个对齐：

| 受影响文件 | 原因 |
|-----------|------|
| **`05_渲染工程/public/assets/ASSET_xx_xx.png`** | 目录导航第 150 行明确规定素材走 symlink。清单里的素材文件新增/重命名——symlink 目标必须指向新文件。不要在 `public/assets/` 里直接存源素材 |
| **`05_渲染工程/asset-manifest.json`** | 记录"ASSET 编号 → 磁盘文件名"的映射表。清单新增或重编号素材——manifest 必须追加或更新对应条目 |
| **`05_渲染工程/src/lib/assets.ts`** | TypeScript 类型表，定义了所有合法 asset ID 的字面量联合类型。清单新增素材编号——类型表必须同步扩展，否则渲染代码里引用新编号会编译报错 |
| **`05_渲染工程/scripts/check-assets.mjs`** | 检查脚本：扫描 manifest 和 public/assets/，确认所有引用文件存在、无一断裂。素材变更后应重跑此脚本 |
| **`04_素材清单/assets/asset_ledger.json`** | 素材台账跟清单是共生关系。清单新增素材——台账要追加 source_url、provider、license、fetched_at 等字段 |
| **端到端管线手册** | 步骤 2 和步骤 3 引用了清单文件路径 |
| **目录导航** | 导航列出了当前素材清单入口 |
