# 省科协科普视频 — Agent 协作规则

> 本文件是 省科协科普视频 项目的工作主规则。Agent 进入项目后第一个读取本文件。

## 语言要求

永远使用中文沟通。

## 工作方式

复杂任务先给计划，再执行。优先最小改动。

## 文档命名规范

所有文件命名使用北京时间（UTC+8），格式：`YYYY-MM-DD_HH-MM_文档名称.md`。文件名的前缀为最后更新时间，每次实质更新都要同步更新文件名和正文时间戳。正文需要显式记录最后更新时间和更新次数。

## 长文档目录规则

超过 80 行的文档必须在标题和正文之间插入带锚点链接的目录。锚点规则：中文标点、括号、斜杠、反引号全部去掉，中文和英文之间不留空格，字母全部小写。章节变化时目录必须同步更新。

## 手递手文档（Handoff）存放位置

Handoff 是写给下一个 Agent 的上下文交接文档，属于项目级工作产物。必须存放到 `tmp/` 目录下（与 `_proposals/` 的提案文档分开存放），不允许放到工作区外的系统 `/tmp`。

## 工程文档写作风格

生成或修改工程文档（工程标准、管线手册、拉片 Prompt 等）时，遵循以下原则，禁止只堆名词和表格：

1. **每个规则都带场景举例。** 不只写"Rotate 用顺时针角度"，还要写"你想让截图旋转 15 度模拟手持拍摄的晃动感就写 Rotate 15° clockwise，别写 css matrix"

2. **先通俗后术语。** 每个概念先用大白话讲清楚这玩意到底是干嘛的、你什么时候会用到它，再给术语名和精确参数。不准一上来就怼一张表格

3. **写反例。** 关键约束必须附一个"如果你写错了会怎样"的反例。比如空间定位那一条："如果你写 left: 1920px，在手机上看就跑飞了——必须写 X: 50%"

4. **不写咨询公司腔调。** 用正常说话的方式写，不要写成 PPT 要点、不要用"赋能""打通""闭环"这种 AI 味的套话

## 强制阅读清单（硬约束）

进入项目后，在任何修改操作之前，Agent 必须先读完以下五个文件：

1. [项目总览](./2026-06-22_15-30_项目总览.md) — 愿景、生产流程、当前进度、缺口
2. [目录导航](./2026-06-22_15-30_目录导航.md) — 目录结构、文件入口、引用维护规则
3. [工程标准：物理字典与素材协议](./2026-06-22_15-30_工程标准_物理字典与素材协议.md) — 运动描述、素材标签、资产协议
4. [端到端管线操作手册](./2026-06-22_15-30_端到端管线操作手册.md) — 步骤 0-5 完整操作指南
5. [文档依赖与影响关系图](./2026-06-22_15-30_文档依赖与影响关系图.md) — 11 条核心依赖链

这五个文件的 IMPORTANT 区块互相引用，形成闭环。第五个文件（依赖关系图）定义了"改 X 会波及哪些 Y"的完整映射。Agent 不能只读了项目总览就开始改文件——必须读完依赖关系图，理解当前状态的依赖网络。

## Blast Radius 行为（硬约束）

当人类提出任何修改意图（无论是改 prompt、改工程标准、新增拉片、还是调整文件结构），Agent 必须按以下流程执行，不得跳过：

1. **对照依赖关系图的 11 条链**，判断本次修改的起点属于哪条链（可能触发多条链）
2. **在聊天里汇报 blast radius**：列出直接影响文件、间接受影响文件、以及明确不会受影响但容易被误改的文件
3. **停住，等你确认。** 不能直接动手改起点文件。你必须看到完整的波及范围并同意后，Agent 才能进入执行
4. **按链条顺序逐文件执行。** 先改起点文件，再按依赖链从上到下改每个受影响文件。每改完一个文件，确认该文件内的关键路径、文件名、版本号不与其他文件矛盾
5. **改完后跑验证。** 至少做：`git diff --name-only` 确认实际修改和计划一致；`rg` 检查关键路径交叉引用没有断裂；确认没有被标注为"不在影响范围内"的文件被误改

## 四个核心文档的职责边界（不可交叉）

| 文档 | 管什么 | 不管什么 |
|------|--------|---------|
| 项目总览 | 愿景、生产流程、当前进度、缺口、下一步 | 不管理目录结构和文件路径索引 |
| 目录导航 | 目录结构、文件入口、Prompt/脚本/视频产物索引、引用维护规则 | 不描述管线步骤和 Agent 指令 |
| 工程标准 | 运动描述约束、素材标签定义、素材资产生命周期、推荐工具 | 不描述"先做哪步后做哪步" |
| 端到端管线手册 | 从文字稿到成片的步骤 0-5 完整操作指南 | 不重复定义物理字典和素材标签 |

同一个信息只能存在于四个文档中的一个。修改时不要把 A 的职责内容写进 B。

## 全局路径约定

本项目中所有带时间戳前缀的文件名（如 `2026-06-22_15-30_项目总览.md`），在各文档内部引用时使用相对路径。依赖关系图中使用简化标识（不携带时间戳，如 `openrouter_segment_analyze.py`），实际精确路径通过目录导航索引获取。

## 结构性变更流程

涉及多文件的结构性变更（Prompt 升级、文档重组、标准修订等），使用 `/doc-proposal` 触发完整流程。

调用链：`/doc-grilling`（追问）→ `/doc-prd`（PRD + 任务拆解）→ `/doc-execute T-NNN`（逐任务执行）。
Skill 定义位于 `.claude/skills/` 目录下。
