本目录管理视频项目的素材全生命周期：从分镜提取素材清单，到 AI 检索筛选，到最终素材。

```
00_文稿  ──┐
           ├→  AI  →  03_分镜脚本/  →  AI  →  04_素材清单/  →  AI+人  →  assets/
标杆视频库 ─┘
共享工具/提示词 ─┘
```

## 目录结构

```
04_素材清单/
├── README.md                          本文件
├── 2026-MM-DD_HH-MM_...素材清单.md     当前素材清单（markdown 驱动，见下）
└── assets/                           当前素材文件（source of truth）
```

## 素材清单 Markdown 格式

素材清单一律使用 markdown 文件驱动，不再使用 JSON。每次试验（不同分镜版本、不同获取策略）生成新的 markdown 文件，旧版移入 `_归档_/视频项目/01_为什么他的豆包更聪明/旧版素材清单/`。

单次素材清单的推荐结构：

```
# 豆包科普 0-30s 素材清单

> 生成时间：YYYY-MM-DD HH:MM
> 对应分镜脚本：03_分镜脚本/xxxx.md

## 素材汇总

| 编号 | 分类 | 描述 | 获取方式 | 状态 |
|------|------|------|---------|------|
| ASSET_01_01 | BROWSER | 豆包 App 首页截图 | Playwright 截图 | [TODO] |
| ASSET_02_01 | CODE | 深灰网格背景 | 代码生成 | [AUTO] |

## 分类说明

- **BROWSER**：浏览器自动化截图（Playwright 等）
- **STOCK**：图库检索的公共素材（Pexels/Unsplash/Pixabay）
- **CODE**：纯代码绘制，无外部依赖
- **AI_GEN**：AI 生成的图片
- **HUMAN**：需人工获取的素材
```

## 素材生命周期

```
[TODO] → AI 改写搜索词 → 并行检索图库 → AI 筛选去重
      → 直接下载到 assets/ASSET_xx_xx.png
      → [DONE]
```

## 文件命名

- 素材清单：`YYYY-MM-DD_HH-MM_豆包科普_0-30s_素材清单.md`
- 素材文件：`ASSET_[镜头序号]_[图层序号].png`（如 ASSET_01_02.png）

## 多次试验约定

每次重新生成分镜或调整素材获取策略时，创建新的素材清单 markdown 文件，旧清单与旧素材移入 `_归档_/视频项目/01_为什么他的豆包更聪明/旧版素材清单/`，保留命名不变以便追溯。
