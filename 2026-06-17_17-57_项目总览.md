## 工作区声明

2026-06-21 起，**省科协科普视频** 升级为独立工作主目录。Agent 进入项目后第一个读取 `./AGENTS.md`，而非上层 `general_chat/` 的全局规则。本目录下的 AGENTS.md 定义了五个文件的强制阅读清单和"任何修改前必先算 blast radius"的硬约束。与提案工作流相关的参考文件（Handoff 文档、Matt Pocock Skills 解析）已迁入本目录。

## 项目愿景

目标是：给定一篇科普文字稿，通过 AI 全链路自动化，生成一部完整的、接近差评君/冲浪Plus 质量水准的科普视频。

人类主要负责两件事：提供科普选题的文字稿，以及最终成片的质量审核与微调。中间的分镜脚本、素材清单、素材获取、程序化渲染代码和成片导出，都尽量交给 AI 按流程完成。

## 全链路生产流程

```text
文字稿（人类提供）
    │
    ▼
第一步：分镜头脚本生成
输入：文字稿 + Few-Shot 拆解样本
输出：逐秒分镜脚本（物理字典描述，不指定具体实现库）
存放：视频项目/.../03_分镜脚本/
    │
    ▼
第二步：素材清单提取
输入：分镜头脚本
输出：结构化素材待办清单（编号/分类/获取指令）
存放：视频项目/.../04_素材清单/
    │
    ▼
第三步：素材自动获取
界面截图 → 浏览器自动化访问并截取
通用素材 → 图库 API 检索 + AI 筛选
代码素材 → 渲染代码直接生成
最终素材：视频项目/.../04_素材清单/assets/
    │
    ▼
第四步：程序化渲染合成
输入：分镜脚本 + 素材 + 库对比测试结论
输出：完整视频（Remotion 渲染）
成片：视频项目/.../06_成片/
```

数据流可以简单理解为：

```text
00_文稿
  ├─ 标杆视频库（Few-Shot 来源）
  └─ 共享工具/提示词（拉片 Prompt）
        ↓
03_分镜脚本
        ↓
04_素材清单
        ↓
04_素材清单/assets
        ↓
05_渲染工程
        ↓
06_成片
```

## Few-Shot 上游：标杆视频拉片

Few-Shot 样本的上游是”标杆视频拉片分析”，当前统一使用 v4 Prompt：
`共享工具/提示词/2026-06-17_15-50_Gemini工业级原片后期解耦拉片分析prompt_v4.txt`

v4 的核心变化是把视频拆成”底层原片事件流”和”后期干预手段”两层分析。具体拉片要求定义见[工程标准：拉片素材标签](./2026-06-17_17-57_工程标准_物理字典与素材协议.md#拉片素材标签)和[场景转场术语](./2026-06-17_17-57_工程标准_物理字典与素材协议.md#场景转场术语)。

已有的 C、D 两个标杆拉片样本是 v2 Prompt 产物，仍可暂时作为 Few-Shot 样本使用。所有新增拉片和返工拉片统一使用 v4 Prompt。

视频压缩参数（用于上传给 Qwen Code 等多模态模型）见 [_参考/视频压缩参考](./_参考/2026-06-22_13-38_视频压缩参考.md)。

## 已知风险

最主要的风险还是偏差逐层累积：拉片阶段模型可能误判复杂图层运动；Few-Shot 阶段模型可能分不清“优秀视觉语法”和“拉片噪音”；渲染阶段再把偏差翻译成代码，最终画面就会越来越偏。

应对方式是继续坚持三件事：拉片时强制原片/后期解耦；分镜时只写标准物理字典；渲染前用素材清单和工程标准做人工检查。
